## Що v0.2 доводить

- [x] Сесія може пройти шлях `ingest -> sleep -> Archive -> MemoryUnit`.
- [x] `core_context_package` і `recall` бачать prompt-facing compact memory, а не host-side JSON-проекцію.
- [x] Fidelity validator перевіряє memory unit по малому evidence pack і не пише в Core.
- [x] Reflection читає validated memory units + Core і створює candidate beliefs, але не пише Core напряму.
- [x] Manual review може промоутити candidate у Core з `source_candidate_id`.
- [x] Candidate з `contradicted_core_fact_ids` переводить старий active Core fact у `contested`, не видаляючи його.
- [x] Forgetting v1 може позначити low-signal unit як `Forgotten`, перебудувати `compact_memory`, і потім повернути unit через `remember_back`.
- [x] Protected units не забуваються навіть при LLM-рекомендації `forget`.
- [x] Telegram host лишається тонким adapter/debug UX: memory-рішення живуть у Rust core.

## Reproducibility Anchor

Комічений deterministic acceptance-test:

```powershell
cargo test -p memory_engine --test living_memory_cycle
```

Тест `crates/memory_engine/tests/living_memory_cycle.rs` проходить повний цикл без live LLM, підставляючи LLM-відповіді через `LlmResponse`:

1. `ingest` змістового повідомлення.
2. Pull-based sleep driver: extraction batch, consolidator batch, `finish_sleep_run`.
3. Archive -> Core bridge.
4. Auto-routed fidelity request і `submit_memory_fidelity_response`.
5. `core_context_package` + `recall`.
6. `reflection_analyze` -> candidate belief.
7. `review_candidate(approved)` -> Core fact.
8. Contradicting candidate -> old Core fact `contested`, new Core fact `active`.
9. `forget_review` -> low-signal unit `Forgotten`.
10. `remember_back` -> unit returns to `ActiveArchive` and parent `compact_memory` is rebuilt.

## What v0.2 Does Not Claim

- Не заявляємо benchmark/accuracy цифри на LongMemEval, RAG-тестах або інших зовнішніх наборах.
- Не заявляємо, що prompt/summary якість вже оптимальна.
- Не заявляємо, що vector storage реалізований.
- Не заявляємо, що forgetting і recall мають unit-level recall counters. v0.2 використовує archive-level recall proxy.
- Не заявляємо, що Core candidate reviewer/formulation pass реалізований. Manual review path працює без нього.

## Deferred After v0.2

- Vector storage як opt-in режим після стабілізації core lifecycle.
- Core candidate reviewer/formulation pass на reasoning-моделі для щільнішого формулювання Core тез.
- Unit-level `recall_count` / `last_recalled_at`.
- Per-scope Core store layout, якщо shared category locks стануть bottleneck.
- Partial sleep/session-tail strategy.
- Benchmark harness і публічні quality claims тільки з reproducibility-anchor.
