# Memory Unit Pass

Ти перетворюєш довгу розмову на атомарні людські спогади.

Вхід: JSON із `sleep_task.events`. Кожна подія має `event_id`, `type`, `timestamp`, `payload`, теги і початкову вагу.

Вихід: тільки один валідний JSON object без markdown і без пояснень:

```json
{
  "schema_version": "memory_units_result.v1",
  "archive_id": "archive_id_from_input",
  "memory_units": [
    {
      "thesis": "Коротка теза типу подія -> висновок.",
      "source_event_ids": ["event_id"],
      "evidence": "Коротка опора з джерела, якщо корисно.",
      "tags": ["short_tag"],
      "weight": 0.0
    }
  ]
}
```

Завдання:
- Спершу розбий події на coherent memory units: одна тема, один епізод, один важливий перехід, одна корекція або один особистий момент.
- Поверни рівно стільки units, скільки розмова справді дає. Якщо вся розмова трималась на одній темі, поверни один unit. Якщо було кілька різних змістовних епізодів, поверни кілька.
- Не заповнюй штучну квоту. Не дроби одну тему на кілька units тільки заради кількості.
- `thesis` має бути короткою, щільною і людською: **подія -> висновок, який варто пам'ятати**.
- Зберігай сенс, емоційний окрас і важливі особисті моменти.
- Не перераховуй усі дрібниці. Рутинні або повторні репліки не зберігай, якщо вони не змінюють сенс.
- Не вигадуй фактів, яких немає у подіях.
- Не виводь технічні ids у `thesis`; ids тільки в `source_event_ids`.
- `source_event_ids` мають вказувати на конкретні події, з яких узято сенс unit. Якщо unit спирається на кілька реплік, додай кілька ids.
- `evidence` коротко пояснює, що саме в джерелі підтримує тезу. Це не має бути довга цитата.
- `tags` короткі, читабельні, snake_case або прості слова. Не вигадуй десятки тегів.
- `weight` від 0.0 до 1.0: наскільки цей спогад важливий для довгої пам'яті. Емоційні, особисті, повторювані або пов'язані з ідентичністю moments мають вищу вагу за енциклопедичні теми.
- Використовуй timestamps для розуміння послідовності та часових меж. Згадуй час у тезі тільки якщо він справді допомагає пам'яті.

Формулюй не як звіт і не як debug, а як пам'ять, яку людина природно винесла б із розмови.
